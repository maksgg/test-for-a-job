# Проєкт "Каталог Товарів"

Сучасний, респонсивний каталог товарів, побудований на Vue 3 та TypeScript.

## Як встановити та запустити проєкт

1. **Клонуйте репозиторій** (якщо застосовно) або перейдіть до директорії проєкту.
2. **Встановіть залежності**:
   ```bash
   pnpm install
   ```
3. **Запустіть сервер для розробки**:
   ```bash
   pnpm dev
   ```
4. **Зберіть проєкт для продакшну**:
   ```bash
   pnpm build
   ```
5. **Запустіть лінтер**:
   ```bash
   pnpm lint
   ```

## Що було реалізовано

- **Адаптивний каталог товарів**: Табличний вигляд для десктопів та спеціалізований вигляд у формі карток (в одну колонку) для мобільних пристроїв.
- **Пошук та фільтрація**: Пошук у реальному часі з дебаунсом, фільтрація за категоріями, наявністю на складі та знижками.
- **Сортування**: Сортування за ціною, рейтингом та назвою.
- **Пагінація**: Функціонал "Load More" (завантажити ще) з обробкою лімітів на юзера оскільки API не підтримує пагінацію в повному обсязі.
- **Система обраного**: Можливість зберігати товари у постійному списку обраного (зберігається в `localStorage` як повні об'єкти).
- **Функціонал порівняння**: Порівняння до 3 товарів у спеціалізованому табличному вигляді із закріпленими лейблами та горизонтальним скролом на мобільних пристроях.
- **Бібліотека спільних компонентів**: Використано власно розроблені реюзабельні компоненти (`VTable`, `VButton`, `VInput`, `VDropDown`, `VCheckbox`, `VLoader`).
- **Сучасний UI/UX**: Професійна кольорова палітра, ховер-ефекти, плавні переходи та піксельно-точне вирівнювання.
- **Навігація клавіатурою**: Повна підтримка взаємодії з додатком через клавіатуру (Enter, Space, стрілки, Escape).
- **Чиста архітектура**: Логіка модульно розподілена у спеціалізовані композабли (`useFavorites`, `useCompare`, `useCatalogSearch`, `useProductsRequest`).

## Що було пропущено

- **Розширені ARIA-лейбли**: Хоча базову підтримку клавіатури та стани фокусу реалізовано, деякі складні підказки для скрінрідерів у транспонованій таблиці порівняння було пропущено.
- **Unit-тести**: Наявна стандартна заготовка проєкту, але специфічні кастомні юніт-тести не додавалися в межах цього завдання.

## Відомі проблеми

- Немає. Проєкт повністю проходить перевірку лінтером та функціонує коректно.

## Що б я покращив за наявності додаткового часу

- **Управління станом**: Інтеграція Pinia для більш надійного управління глобальним станом у разі зростання складності додатка.
- **UI/UX**: Оскільки готового макету не має для кращого візуалу потрібно більше часу.
- **Refactoring**: Оскільки багато що писалось за допомоги ШІ, зокрема 90% стилізації css впевнений що присутньо багато сміттєвих css властивостей які потребують рефакторингу. Також банально потрібно детально провести тестування на наявність багів та не відповідностей функціоналу.
- **Add features**: До кінця не зрозумів як саме потрібно фільтрувати по категоріям, але певна ідея в мене закралась в головві, але на це потрібно більше часу оскільки прийдеться рефакторити трохи реюзабл компоненти зокрема VDropDown.vue.
